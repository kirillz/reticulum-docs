    Основы RNS¶

    Что такое Reticulum?¶

    Reticulum (RNS) — это криптографически защищенный сетевой стек для построения оверлейных mesh-сетей любой топологии. Это не готовое приложение, а протокол, который можно использовать как фундамент для создания собственных решений.

    Ключевые особенности¶

    Особенность Описание
    Независимость от транспорта Работает поверх любого канала: TCP/IP, LoRa, WiFi, Serial, лазер, радиомодемы и даже экзотические транспорты
    Сквозное шифрование Все соединения защищены по умолчанию, без необходимости дополнительной настройки
    Самоорганизация Узлы автоматически обнаруживают друг друга и строят маршруты без центрального сервера
    Универсальность Поверх RNS можно запускать любые приложения: чаты, файлообменники, удалённые терминалы, веб-сервисы

    Как это работает?¶

    Reticulum создаёт оверлейную сеть поверх существующих физических каналов связи. Каждый узел в сети имеет уникальный криптографический идентификатор, а сообщения маршрутизируются между узлами с гарантией доставки и целостности данных.

    ┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   Узел A    │ ◄──► │   Узел B    │ ◄──► │   Узел C    │
│  (LoRa)     │      │  (WiFi)     │      │ (Ethernet)  │
└─────────────┘      └─────────────┘      └─────────────┘
        ▲                                        │
        │            Reticulum Overlay           │
        └────────────────────────────────────────┘

    Инструменты RNS¶

    После изучения основ перейдите к практическим утилитам:

    Утилита Назначение
    rnsd Демон Reticulum — фоновая служба для работы сети
    rnstatus Просмотр статуса сети и подключений
    rnid Генерация и управление идентификаторами
    rnpath Диагностика маршрутов между узлами
    rnprobe Проверка доступности узлов и каналов
    rncp Копирование файлов через сеть Reticulum
    rnx Удалённое выполнение команд на других узлах
    rnodeconf Настройка TNC и радиомодемов RNode
