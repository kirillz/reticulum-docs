# rnsd

**rnsd** — демон сетевого стека Reticulum. Фоновая служба, обеспечивающая работу сети Reticulum на вашем узле.

Запуск Reticulum в качестве службы очень прост — достаточно выполнить входящую в комплект команду `rnsd`. Когда `rnsd` запущен, он поддерживает все настроенные интерфейсы активными, обрабатывает транспорт (если он включён) и позволяет любым другим программам немедленно использовать сеть Reticulum, для которой он настроен.

На одном компьютере можно даже запустить несколько экземпляров `rnsd` с разными конфигурациями.

---

## Примеры использования

**Запустить rnsd:**

```bash
$ rnsd

[2023-08-18 17:59:56] [Notice] Started rnsd version 1.1.4
```

**Запустить rnsd в режиме службы**, чтобы весь вывод логов направлялся непосредственно в файл:

```bash
$ rnsd -s
```

**Сгенерировать подробный пример конфигурации** с описанием всех доступных опций и примерами настройки интерфейсов:

```bash
$ rnsd --exampleconfig
```

---

## Все опции командной строки (переведено для понимания)

```
usage: rnsd.py [-h] [--config CONFIG] [-v] [-q] [-s] [--exampleconfig] [--version]

Демон сетевого стека Reticulum

options:
  -h, --help        показать эту справку и выйти
  --config CONFIG   путь к альтернативному каталогу конфигурации Reticulum
  -v, --verbose     подробный вывод
  -q, --quiet       тихий режим (минимум сообщений)
  -s, --service     rnsd запущен как служба и должен логировать в файл
  -i, --interactive перейти в интерактивную оболочку после инициализации
  --exampleconfig   вывести подробный пример конфигурации в stdout и выйти
  --version         показать номер версии программы и выйти
```
