## lxmd

Утилита `lxmd` используется для запуска Propagation Node (узла распространения). Это узел, который хранит сообщения для пользователей, которые сейчас не в сети (аналог почтового сервера).

`lxmd` — это демон узла распространения сообщений LXMF (LXMF Propagation Node).Он позволяет хранить и пересылать сообщения для пользователей, которые находятся оффлайн или имеют неустойчивое соединение.

#### Использование
Запуск узла распространения:

`lxmd`
Основные аргументы
```
-c, --config — Путь к директории конфигурации.
--example-config — Вывести пример файла конфигурации.
-i, --on-interface — Автоматически привязать узел к указанному интерфейсу RNS.
```
#### Принцип работы
Когда пользователь Nomad Network не в сети, отправленное ему сообщение попадает на ближайший Propagation Node (если он доступен). Когда пользователь выходит в сеть, он забирает (sync) сообщения с узла.

Для работы lxmd требует наличия запущенного Reticulum (`rnsd`).
